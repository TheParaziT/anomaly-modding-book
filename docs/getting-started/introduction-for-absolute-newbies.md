## Визуальная часть

Формат, в котором храниться визуал это ogf и object. 
Так же важным знанием будет то, что это Динамический Обьект. 

Динамический обьект.
Это в глосарии есть

Чуть подробнее.
Ogf - скомпилированный (удалены ненужные блоки данных, вершины и оптимизация). Используется напрямую в игре
Object - исходный формат моделей, для работы в SDK.

### Surface

Копнем глубже,
У модели должен быть surface, в который входит

### Game Material 
Определяет свойства материала (Упругость, проникновение звука, плавучесть, пробивную способность и прочее)

#### Engine Shader
Как выглядит. Самое важное, для динамических обьектов требуются только шейдеры для динамических обьектов! 

#### Compile Shader
Нужен для статической геометрии. 

## Animations | Skeleton

*.bones

Содержит описание костей скелета

*.anm

Анимация камеры и путей обьектов (аномалий, вертолетов и т.п.)

Идем далее, что умеет сталкер? Бегать, ходить, стрелять и прочее.
Все это анимации. Так же как и с моделями, тут есть две версии форматов файлов.

Skl - исходный формат для работы в sdk

Omf - скомпилированная библиотека анимаций

Что дальше? Ум и логика.
